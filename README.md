# 🌿 作業抽查管理系統 (獎懲建議表自動化模組)

本專案是一個專為學校行政與教師設計的「作業抽查與獎懲單自動化生成系統」。透過前端網頁與 Google Apps Script (GAS) 的深度整合，解決了過去手動登記未交作業、人工分班、以及逐一列印獎懲單的繁瑣流程。

## ✨ 核心特色 (Features)

* **🚀 智慧貼上與自動分班**：支援直接從 Excel 複製「學號/班級座號 (5碼)」與「姓名」，系統會自動解析出班級與座號，並瞬間完成分班。
* **➕ 重複名單自動累加**：若同一名學生有多筆缺交紀錄，系統會自動將其合併為一筆，並**自動累加警告支數**。
* **⚙️ 雙軌設定模式 (全域/個別)**：提供全域統一設定（學年度、日期、抽查類別、作業標題），並支援一鍵解除同步，讓特定班級保有獨立的事由設定。
* **🖨️ 完美的 PDF 產出與批次下載**：
    * 自動將標題與事由完美分離，符合正式公文格式。
    * 強制套用「橫印 (Landscape)」與「窄邊界」，完美消滅跨頁孤兒行。
    * 一鍵批次下載當頁所有班級的 PDF，直接全選列印。
* **🛡️ 防漏單排隊機制**：針對 Google 伺服器的限制，內建 Async/Await 非同步排隊與 1.5 秒延遲機制，保證大量班級產出時「絕對不漏單」。
* **📄 歷史紀錄分頁引擎**：自動記憶過去的產出紀錄，並內建分頁功能（每頁 10 筆），保持介面清爽不卡頓。

---

## 🛠️ 系統架構 (Architecture)

本系統採「前後端分離」架構：
1.  **前端 (Frontend)**：`index.html` (純 HTML, CSS, Vanilla JS)。負責介面互動、名單智慧解析、資料分裝與防漏單排隊發送。
2.  **後端 (Backend)**：`Code.gs` (Google Apps Script)。接收前端陣列，進行 PDF 表單繪製、資料庫極速批次寫入，並回傳歷史紀錄。
3.  **資料庫 (Database)**：Google 試算表（包含「學生名冊」與「作業抽查名單」工作表）。

## 🚀 使用方式 (Usage)

### 1. 部署後端 (GAS)
1. 將 `Code.gs` 的內容貼入您的 Google Apps Script 專案中。
2. 確認程式碼頂端的 `TEMPLATE_ID` 已替換為您正確的 Google 試算表範本 ID。
3. 點擊右上角 **「部署」** > **「管理部署作業」**。
4. 點擊 **「✏️編輯」**，版本選擇 **「建立新版本」**，權限設為「所有人」。
5. 按下部署後，複製產生的 **網頁應用程式 URL**。

### 2. 設定前端 (HTML)
1. 打開 `index.html`。
2. 找到 `<script>` 區塊開頭的 `GAS_URL` 變數。
3. 將剛才複製的 URL 貼上替換：
   ```javascript
   const GAS_URL = 'https://script.google.com/macros/s/您的專屬ID/exec'; 
   ```
4. 儲存檔案，直接用瀏覽器打開 `index.html` 即可開始使用！

### 3. 日常操作流程
1. 打開網頁，設定最上方的「全域表單設定」（學年度、抽查類別、作業標題）。
2. 點擊 **「📋 貼上名單」**，將 Excel 中的未交名單（5碼學號 + 姓名）直接貼入，點擊解析。
3. 確認各班卡片資料無誤後，點擊最下方的 **「💾 儲存並產出所有正式獎懲表」**。
4. 等待進度條跑完（系統會自動防漏單排隊）。
5. 在右側「產出紀錄」面板，點擊 **「全選本頁」** ➔ **「🖨️ 批次下載打勾的 PDF」**。
6. 打開下載好的 PDF，列印！

## 📅 版本歷程 (Version History)

### 🌟 前端介面 (HTML/JS)
* **v2.6.0 (終極正式版)**：微調 UI 文字與使用者體驗，整合所有功能。
* **v2.5.0 (極致排版版)**：將「大標題（學年度+類別）」與「事由後綴」分離送出，解決表格內容過度擁擠的問題。
* **v2.4.0 (歷史紀錄分頁版)**：導入前端分頁引擎（每頁 10 筆），解決歷史紀錄過多導致的版面過長與批次下載當機問題。
* **v2.3.1 (防漏單排隊版)**：導入 `Async/Await` 與 `Sleep(1500ms)` 機制，解決 Google API 頻率限制導致的隨機漏單問題。
* **v2.3.0 (窄邊界防跨頁版)**：在 PDF 下載連結寫入 `portrait=false` (橫印) 與 `margin=0.2` (縮減邊界) 參數，完美解決簽章欄跨頁問題。
* **v2.0.0 (雙軌設定版)**：新增「同步套用至所有班級」開關，支援全域統一設定或個別班級獨立設定（如不同班級缺交不同科目）。
* **v1.8.0 ~ v1.9.0**：新增學年度、抽查類別下拉選單，提升公文正式度。
* **v1.5.0 ~ v1.6.0 (全域智慧分班與累加版)**：重大突破！支援貼上跨班級混和名單，系統自動根據 5 碼學號切分班級，並**自動合併重複學生、累加警告支數**。
* **v1.3.0 ~ v1.4.0**：支援從 Excel 複製貼上，自動過濾中文姓名並抓取數字。
* **v1.0.0 ~ v1.2.0**：建立森林系 UI，實作基礎的新增與警告支數自訂功能。

### 🌟 後端 API (Code.gs)
* **v8.5.0**：完全解除前端歷史紀錄顯示上限；修復 `historyOrder` 排序 Bug（確保重複產出的班級能正確排到最頂端）；修正起始抓取列數（從 Row 1 開始）。
* **v8.4.0**：支援接收 `docTitle` 變數，動態替換 PDF 最上方的「(公告)」大標題。
* **v8.2.0 (極速優化版)**：
    * **修復跨頁污染**：改為先複製所有空白分頁再填入資料，解決第 2 頁殘留第 1 頁資料的嚴重 Bug。
